# MTS_MultAgent - Project Brief

## Проект
Многоагентная система для автоматизации анализа корпоративных данных и публикации отчетов в Confluence.

## Основная цель
Создать интеллектуальную систему, которая автоматически:
1. Читает задачи и протоколы совещаний из Jira
2. Анализирует контекст по ключевым фразам (названиям проектов)
3. Извлекает данные из Excel файлов на основе анализа
4. Публикует результаты в Confluence с таблицами и комментариями
5. Выполняет сравнительный анализ и публикует выводы

## Бизнес-проблема
Ручная обработка корпоративных данных требует значительного времени и склонна к ошибкам. Сотрудники тратят часы на:
- Поиск релевантной информации в протоколах совещаний
- Извлечение данных из Excel отчетов
- Формирование отчетов в Confluence
- Сравнительный анализ данных

## Решение
Автоматизированная система из 5 специализированных агентов:
- **JiraAgent**: чтение данных из Jira API
- **ContextAnalyzer**: анализ контента и поиск по ключевым фразам
- **ExcelAgent**: работа с Excel файлами
- **ConfluenceAgent**: публикация в Confluence
- **ComparisonAgent**: сравнительный анализ

## Технологический стек
- **Язык**: Python 3.9+
- **Асинхронность**: asyncio, aiohttp
- **Excel**: pandas, openpyxl
- **CLI**: click
- **Web**: FastAPI (второй этап)
- **Конфигурация**: python-dotenv

## Ключевые требования
1. **Персональные токены** для доступа к Jira и Confluence
2. **Асинхронная работа** агентов
3. **CLI интерфейс** для запуска процессов
4. **Web интерфейс** с аутентификацией (второй этап)
5. **Масштабируемость** и расширяемость

## Ожидаемый результат
- Сокращение времени обработки данных на 80%
- Повышение точности анализа
- Автоматизация рутинных операций
- Единая система для работы с корпоративными данными

## Статус проекта
**Текущий этап**: Начальный - создание архитектуры и базовой структуры
**Следующий этап**: Реализация основных агентов и CLI интерфейса
