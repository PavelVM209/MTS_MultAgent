# MTS_MultAgent - Project Brief

## Проект
Интеллектуальная многоагентная система с LLM-интеграцией для автоматизации анализа корпоративных данных и публикации отчетов в Confluence.

## Основная цель
Создать **самообучающуюся систему** с итеративным улучшением результатов, которая автоматически:
1. **Интеллектуально анализирует** задачи и протоколы совещаний из Jira через LLM
2. **Динамически формирует запросы** к Excel файлам на основе контекста без hardcoded паттернов
3. **Извлекает и анализирует данные** с адаптивным улучшением качества
4. **Публикует результаты** в Confluence с таблицами и интеллектуальными выводами
5. **Выполняет итеративное сравнение** с самооценкой и корректировкой до достижения оптимального качества

## 🧠 НОВЫЙ ПОДХОД - LLM-CENTRIC ARCHITECTURE

### Ключевое отличие от традиционных систем:
- **Никаких hardcoded паттернов** - все решения через LLM
- **Итеративное самоулучшение** - система улучшает результаты до достижения качества
- **Динамическая адаптация** - запросы корректируются на основе анализа результатов
- **Интеллектуальная валидация** - оценка качества через LLM на каждом шаге

## Бизнес-проблема
Традиционные системы автоматизации не способны:
- **Адаптироваться к уникальным бизнес-контекстам** без переписывания кода
- **Самоулучшаться** при получении неудовлетворительных результатов
- **Понимать семантику** данных без предустановленных правил
- **Генерировать контекстуальные выводы** на основе анализа

## 🔄 ИНТЕЛЛЕКТУАЛЬНОЕ РЕШЕНИЕ

### Архитектура с итеративным улучшением:
- **JiraAgent**: чтение данных из Jira API
- **ContextAnalyzer (LLM-driven + Self-Evaluation)**: интеллектуальный анализ контекста с итеративным улучшением запросов
- **ExcelAgent (LLM-guided)**: обработка Excel с динамическими запросами
- **ComparisonAgent (LLM-driven + Adaptive)**: анализ с адаптивным улучшением результатов
- **ConfluenceAgent**: публикация в Confluence

### 🔄 Итеративный цикл улучшения:
```
Context → LLM Analysis → Query Excel → Result Evaluation → Quality Check → 
└─ If Quality < Threshold → Refine Query → Repeat │
└─ If Quality ≥ Threshold → Generate Insights → Publish
```

## 🚀 ИННОВАЦИОННЫЕ АСПЕКТЫ

### LLM-Driven Column Mapping:
```
Traditional: IF column contains "revenue" THEN map to financial_metrics
LLM-Driven: Analyze column semantics → Understand business context → Map dynamically
```

### Intelligent Query Refinement:
```
Iteration 1: "Get financial data for companies"
Result: Basic revenue numbers
Evaluation: "Missing profitability analysis"
Iteration 2: "Get financial data including revenue, costs, profitability metrics"
Result: Comprehensive financial analysis
Evaluation: "Quality ≥ 95% - Accept"
```

### Quality Metrics System:
- **Relevance Score** (0-100%): соответствие контексту
- **Completeness Score** (0-100%): полнота анализа
- **Accuracy Score** (0-100%): точность извлечения
- **Overall Quality** (0-100%): интегральная оценка
