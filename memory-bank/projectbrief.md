# MTS MultAgent Project Brief

## 🎯 Основная цель проекта

Создать интеллектуальную многоагентную систему мониторинга сотрудников MTS с использованием LLM-анализа для автоматизации рутинных процессов управления командой и генерации отчетов.

## 📋 Исходные требования

**Первоначальная задача:** Использовать существующие наработки, но с измененной архитектурой:

1. **Jira Analysis Agent** - ежедневно анализирует задачи из Jira, отслеживает прогресс по сотрудникам, сохраняет отчеты
2. **Meeting Analysis Agent** - ежедневно анализирует протоколы собраний, определяет прогресс по сотрудникам, сохраняет отчеты  
3. **Weekly Reports Agent** - еженедельно (пятница вечер) делает комплексный анализ по каждому сотруднику, публикует в Confluence
4. **Orchestrator Agent** - оркестрирует все процессы, проверяет качество отчетов, отправляет на доработку при необходимости

## 🏗️ Архитектурные изменения

В отличие от первоначального проекта с фокусом на Excel-анализ, система переориентирована на:

- **Анализ корпоративных данных** (Jira + Confluence + протоколы)
- **Мониторинг производительности сотрудников** 
- **Автоматическую генерацию отчетов для руководства**
- **Контроль качества с использованием LLM**

## 🎬 Бизнес-проблема

Руководителям необходимо ежедневно отслеживать:
- Производительность команды по задачам Jira
- Активность сотрудников на совещаниях  
- Комплексную картину эффективности каждый сотрудника
- Выявлять проблемные зоны и.success-кейсы

Текущий процесс требует ручного сбора данных и анализа, что занимает много времени и может быть субъективным.

## 🔧 Технологическое решение

### Core Components:

1. **TaskAnalyzerAgent** - Анализ Jira задач
   - Подключение к MTS Jira API
   - Анализ статусов, приоритетов, сроков
   - Расчет метрик производительности

2. **MeetingAnalyzerAgent** - Анализ протоколов
   - Обработка TXT/DOCX/PDF файлов
   - Извлечение участников и активности
   - Оценка вовлеченности

3. **WeeklyReportsAgent** - Еженедельная аналитика
   - Комплексный анализ по сотрудникам
   - Генерация выводов и рекомендаций
   - Публикация в Confluence

4. **QualityValidatorAgent** - Контроль качества
   - Проверка полноты данных
   - Валидация выводов через LLM
   - Запрос на доработку при необходимости

5. **EmployeeMonitoringOrchestrator** - Оркестрация
   - Управление workflow
   - Координация агентов
   - Обработка ошибок

6. **EmployeeMonitoringScheduler** - Планирование
   - Cron-like расписание
   - Параллельное выполнение
   - Обработка зависимостей
