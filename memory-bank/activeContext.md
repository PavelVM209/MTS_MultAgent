# Active Context

## Текущий фокус работы (Апрель 2026)

Проект MTS_MultAgent находится в фазе подготовки к инкрементальному сравнению изменений по новым протоколам/задачам Jira на основе последовательности запусков (run). Выполнен переход на run-based артефакты (`reports/runs`, `reports/latest`) и введена инкрементальная обработка протоколов по хэшу через `ProcessingIndex`.

ADR: `docs/adr/0001-run-artifacts-and-incremental-processing-index.md`

## Полученные знания и инсайты проекта

### Технические инсайты
1. **LLM client critical importance** - без него система не работает
2. **Regex patterns need careful testing** с реальными данными
3. **Structured data extraction требует надежных fallbacks**
4. **API integration needs proper error handling**

### Архитектурные решения
1. **Three-stage analysis** significantly improves quality
2. **File-based storage** подходит для инкрементального анализа
3. **Modular design** упрощает отладку и расширение
4. **Type hints и dataclasses** улучшают поддерживаемость

### Проблемные зоны и решения
1. **Employee name extraction** - решено через improved версии
2. **Jira API доступ** - полностью валидирован и готов к production
3. **Комментарии в Jira** - требуют отдельных API вызовов, созданы стратегии оптимизации
4. **Performance Meeting Analyzer** - критическая проблема (83 минуты обработки)
5. **Интеграция агентов** - восстановлена (Weekly Reports читает run-артефакты `reports/runs/*`)

### Новые архитектурные инсайты (апрель 2026)
1. **Jira интеграция критически важна** для инкрементального анализа
2. **BaselineABLE подход** - система готова к сравнению новых данных с существующим анализом
3. **Производительность LLM** - основной bottleneck в real-time обработке
4. **Комментарии как источник данных** - требуют стратегического подхода к получению
5. **Масштабирование системы** - текущая архитектура выдержит рост до 50+ сотрудников
