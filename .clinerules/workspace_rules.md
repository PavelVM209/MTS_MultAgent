# Руководство по проекту

всегда пиши на русском языке

## Требования к документации

-   Обновлять соответствующую документацию в /docs при изменении функций
-   Поддерживать README.md в синхронизации с новыми возможностями
-   Вести записи об изменениях в CHANGELOG.md

## Записи архитектурных решений

Создавать ADR в /docs/adr для:

-   Значительных изменений зависимостей
-   Изменений архитектурных паттернов
-   Новых паттернов интеграции
-   Изменений схемы базы данных
    Следовать шаблону в /docs/adr/template.md

## Стиль кода и паттерны

-   Генерировать клиенты API с помощью OpenAPI Generator
-   Использовать шаблон TypeScript axios
-   Размещать сгенерированный код в /src/generated
-   Предпочитать композицию наследованию
-   Использовать паттерн репозитория для доступа к данным
-   Следовать паттерну обработки ошибок в /src/utils/errors.ts

## Стандарты тестирования

-   Обязательны модульные тесты для бизнес-логики
-   Интеграционные тесты для API endpoints
-   E2E тесты для критических пользовательских потоков
